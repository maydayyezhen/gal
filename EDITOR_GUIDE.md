# 剧本编辑器使用手册 📝

## 🎯 编辑器特点

- ✅ **可视化编辑** - 无需手写代码，点击即可编辑
- ✅ **资源管理** - 拖拽上传图片和音频
- ✅ **实时预览** - 编辑立即生效
- ✅ **自动保存** - 数据保存在浏览器本地
- ✅ **一键导出** - 生成标准的章节文件

## 🚀 快速开始

### 1. 打开编辑器

双击 `editor.html` 文件，在浏览器中打开编辑器。

**编辑器会自动：**
- ✅ 加载项目中已有的章节（`chapters/chapter1.js` 等）
- ✅ 显示章节的场景和对话
- ✅ 列出章节中使用的资源
- ✅ 自动选中第一个章节，方便直接编辑

**如果看到章节列表：**
- 点击任意章节开始编辑
- 左上角会显示"已加载 X 个项目章节"

**如果章节列表是空的：**
- 说明项目中没有章节文件
- 点击"+ 新建章节"创建第一个章节

### 2. 编辑现有章节

1. 在左侧列表中点击要编辑的章节
2. 章节信息会自动加载
3. 在"📝 场景编辑"标签页查看和编辑场景
4. 在"⚙️ 章节设置"标签页修改章节信息
5. 在"🎨 资源管理"标签页查看使用的资源

### 3. 创建新章节（如果需要）

1. 点击左侧边栏的 **"+ 新建章节"** 按钮
2. 新章节会自动创建并选中
3. 点击顶部的 **"⚙️ 章节设置"** 标签页
4. 填写章节信息：
   - 章节号：1
   - 章节标题：第一章 你的故事
   - 作者：你的名字

### 3. 添加场景

1. 回到 **"📝 场景编辑"** 标签页
2. 点击 **"+ 添加新场景"** 按钮
3. 在弹出的编辑窗口中填写场景信息

## 📖 场景编辑指南

### 场景基本信息

**场景ID** (必填)
- 格式：`scene_ch1_场景名`
- 例如：`scene_ch1_classroom`, `scene_ch1_rooftop`
- 每个场景ID必须唯一

**背景图ID** (可选)
- 格式：`bg_背景名`
- 例如：`bg_school`, `bg_park`
- 对应 `images/backgrounds/` 目录下的图片

**背景音乐ID** (可选)
- 格式：`bgm_音乐名`
- 例如：`bgm_daily`, `bgm_mystery`
- 对应 `audios/bgm/` 目录下的音乐

### 对话内容编辑

**格式规则：**
```
说话者|对话内容
```

**示例：**
```
阳菜|早上好！
夜|早安。
|这是旁白，不写说话者
阳菜|今天天气真好呢~
```

**注意事项：**
- 每行一条对话
- 旁白留空说话者（只写 `|对话内容`）
- 用 `|` 分隔说话者和对话

### 选择分支

**格式规则：**
```
选项文本>> 下一场景ID
```

**示例：**
```
同意她的请求>> scene_ch1_agree
拒绝>> scene_ch1_refuse
```

**注意事项：**
- 每行一个选项
- 用 `>>` 连接选项和场景ID
- 场景ID要对应实际存在的场景

### 自动跳转

如果场景没有选择分支，会自动跳转到下一个场景。

**填写方式：**
- 留空：结束游戏
- 填写场景ID：自动跳转到指定场景

## 🎨 资源管理

### 导入图片

1. 切换到 **"🎨 资源管理"** 标签页
2. 点击上传区域或直接拖拽图片文件
3. 支持格式：PNG, JPG, JPEG

**图片命名规则：**
- 背景图：`bg_场景名.png`
  - 例如：`bg_school.png`, `bg_park.png`
  - 会自动放入 `images/backgrounds/`
- 角色立绘：`char_角色名_表情.png`
  - 例如：`char_hina_happy.png`, `char_sora_normal.png`
  - 会自动放入 `images/characters/`
- 其他图片：任意名称
  - 会放入 `images/others/`

### 导入音频

**音频命名规则：**
- 背景音乐：`bgm_音乐名.mp3`
  - 例如：`bgm_daily.mp3`, `bgm_battle.mp3`
  - 会自动放入 `audios/bgm/`
- 音效：`se_效果名.mp3`
  - 例如：`se_bell.mp3`, `se_click.mp3`
  - 会自动放入 `audios/se/`
- 语音：`voice_角色名_编号.mp3`
  - 例如：`voice_hina_01.mp3`
  - 会自动放入 `audios/voices/`

## 💾 保存和导出

### 自动保存

编辑器会自动将数据保存到浏览器本地存储，下次打开会自动加载。

### 手动保存

点击左侧边栏的 **"💾 保存全部"** 按钮：
- 保存到本地存储
- 下载所有章节的 `.js` 文件到浏览器下载目录

### 数据导出：

点击 **"📦 导出项目"** 按钮：
- 下载所有章节文件（`chapter1.js`, `chapter2.js` 等）
- 下载资源放置说明文件
- 将下载的文件放入项目对应目录

**放置位置：**
```
game_project/
├── chapters/
│   ├── chapter1.js    ← 下载的章节文件放这里
│   ├── chapter2.js
│   └── ...
```

### 重新加载项目章节

如果你手动修改了 `chapters/` 目录下的文件，可以重新加载：

1. 点击 **"🔄 重新加载项目章节"** 按钮
2. 确认重新加载（会覆盖编辑器中的修改）
3. **刷新浏览器页面**（F5）让新文件生效
4. 编辑器会重新读取所有章节文件

**注意：** 重新加载会用项目文件覆盖编辑器中未保存的修改！

## 🔧 实用技巧

### 1. 快速复制场景

1. 找到要复制的场景
2. 点击"✏️ 编辑"
3. 复制所有内容
4. 创建新场景并粘贴
5. 修改场景ID和内容

### 2. 场景排序

使用场景卡片上的 **↑** 和 **↓** 按钮调整场景顺序。

### 3. 批量编辑对话

对话内容支持多行编辑：
- 复制到文本编辑器中批量修改
- 粘贴回来即可

### 4. 检查场景连接

确保每个选择分支和自动跳转的场景ID都存在，否则游戏会报错。

## ⚠️ 注意事项

### 数据安全

- 编辑器数据保存在浏览器本地
- **清除浏览器数据会丢失编辑内容**
- 建议定期点击"保存全部"下载备份

### 资源文件

- 编辑器只是管理资源信息
- **实际文件需要手动放入项目目录**
- 或使用"导出项目"功能查看放置说明

### 场景ID规范

- 同一章节内场景ID不能重复
- 建议使用 `scene_ch章节号_场景名` 格式
- 例如：`scene_ch1_classroom`, `scene_ch2_forest`

### 浏览器兼容性

- 推荐使用 Chrome / Edge / Firefox 最新版
- Safari 可能有部分功能限制

## 🐛 常见问题

**Q: 打开编辑器后看不到我的章节？**

A: 
1. 确认 `chapters/chapter1.js` 等文件存在
2. 刷新浏览器页面（F5）
3. 查看浏览器控制台（F12）是否有错误
4. 确认章节文件格式正确（使用 `chapterLoader.registerChapter`）

**Q: 我修改了chapter1.js文件，编辑器没显示新内容？**

A: 
1. 点击"🔄 重新加载项目章节"按钮
2. **刷新浏览器页面（F5）**
3. 编辑器会重新读取文件

**Q: 导入的资源在游戏中不显示？**

A: 编辑器只记录资源信息，实际文件需要手动复制到项目目录。点击"导出项目"查看详细的资源放置说明。

**Q: 编辑的内容丢失了？**

A: 数据保存在浏览器本地存储中。如果清除了浏览器数据，内容会丢失。建议定期使用"保存全部"功能下载备份。

**Q: 如何测试编辑的章节？**

A: 
1. 点击"保存全部"下载章节文件
2. 将文件放入 `chapters/` 目录
3. 打开 `game.html` 或 `index.html` 测试

**Q: 可以同时编辑多个章节吗？**

A: 可以。创建多个章节后，在左侧列表中切换即可。

**Q: 如何删除章节？**

A: 目前版本暂不支持删除章节。如不需要某个章节，导出时不要使用该文件即可。

## 📚 进阶使用

### 自定义场景效果

编辑器目前只支持基础场景编辑。如需添加特效、角色动画等高级功能，需要：

1. 先在编辑器中创建基础场景
2. 导出章节文件
3. 在文本编辑器中手动添加高级属性
4. 参考 `chapters/chapter1.js` 中的示例

### 多人协作

如果多人协作编辑：

1. 每人负责不同章节
2. 分别使用编辑器创建
3. 导出后合并到项目中

### 版本控制

建议使用 Git 管理项目：
```bash
git init
git add .
git commit -m "添加第1章"
```
