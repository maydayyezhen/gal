# 重构验收清单

本文档用于每次结构整理、路径迁移或模块拆分后的人工验收。

当前项目是纯静态 Web 项目，没有自动化测试框架，因此每次重构后至少需要完成以下手动检查。

## 1. 基础启动检查

### 1.1 标题页

打开：

```text
index.html
```

检查：

- 页面能正常打开。
- 标题背景显示正常。
- 标题按钮显示正常。
- “开始游戏”按钮可点击。
- “继续游戏”按钮在有存档时显示合理。
- 标题音乐开关不报错。

### 1.2 游戏页

打开：

```text
game.html
```

检查：

- Loading 正常出现并消失。
- 第一章第一场景能正常进入。
- 背景能显示。
- 对话框能显示。
- 点击能推进对白。

### 1.3 编辑器页

打开：

```text
editor.html
```

检查：

- 编辑器页面能正常打开。
- 左侧章节列表能显示。
- 能读取已有章节。
- 点击章节后能显示场景列表。
- 场景卡片样式正常。

## 2. 游戏流程检查

检查：

- 对白点击推进正常。
- 打字机效果正常。
- 旁白显示正常。
- 说话人名字显示正常。
- AUTO 按钮可用。
- SKIP 按钮可用。
- LOG 对话记录可打开。
- 设置面板可打开和关闭。
- 返回标题页按钮可用。

## 3. 场景与分支检查

检查：

- 背景切换正常。
- BGM 切换正常。
- 角色立绘显示正常。
- 角色说话亮暗正常。
- 选择分支出现正常。
- 点击分支后能跳转到对应场景。
- `autoNext` 自动跳转正常。

## 4. 特效与动画检查

检查：

- 角色普通动画正常，如 bounce / heartbeat / hurt。
- 角色待机动画正常。
- flash / shake 等基础特效正常。
- sleep / eyeClose / eyeOpen 等氛围特效正常。
- spell / beam / hit 等局部特效正常。
- 粒子效果正常。
- 道具显示和隐藏正常。

## 5. 存档读档检查

检查：

- SAVE 快速存档可用。
- 设置面板里的快速存档可用。
- 读档后能回到正确场景和对白位置。
- 返回标题页前自动存档不报错。
- 从标题页继续游戏可用。

## 6. 编辑器功能检查

检查：

- 能选择章节。
- 能新建章节。
- 能添加场景。
- 能编辑场景 ID。
- 能选择背景。
- 能选择 BGM。
- 能编辑初始舞台角色。
- 能编辑对白。
- 能配置选择分支。
- 能保存到 localStorage。
- 能导出章节文件。

## 7. Playtest 检查

在编辑器中点击场景或对白 Playtest。

检查：

- 能打开 `game.html?scene=xxx`。
- 能跳到对应场景。
- 如果带 `line` 参数，能跳到对应对白附近。
- Playtest 不影响正式游戏入口。

## 8. 路径迁移专项检查

如果本次改动移动了 JS、CSS 或资源路径，额外检查：

- 浏览器控制台没有 404。
- 浏览器控制台没有 `SCRIPT_DATA 未定义`。
- 浏览器控制台没有 `GG.xxx is not a function`。
- 所有 `<script src="...">` 路径正确。
- 所有 `<link rel="stylesheet" href="...">` 路径正确。
- 资源图片路径仍能加载。
- 音频路径或 generated 音频仍可用。

## 9. 最低验收标准

如果时间不够，至少完成：

```text
1. index.html 能进入游戏
2. game.html 能播放第一章前几个场景
3. 选择分支能点击并跳转
4. editor.html 能打开并读取 chapter1
5. Playtest 能跳转到指定场景
```

这 5 项通过，才可以认为本次整理没有破坏主链路。
