# 测试过程 + 预期/实际结果（验收点）

版本：v1.6.3_M01_S01_D01_D02_D03_editor_preview_fullscreen（Preview Resize Patch）
日期：2026-02-08

## 变更摘要
1) 右侧预览框增加“等比适配（16:9）”的预览视口（居中 + 黑边），避免宽度拉满导致画面被压扁、纵向只显示一小条。
2) 增加可拖拽分隔条：
   - 宽屏（左右布局）：拖动分隔条可调整预览区域宽度
   - 窄屏（上下布局）：拖动分隔条可调整预览区域高度
3) 预览区域尺寸写入 localStorage（刷新/重开场景编辑器后保留比例）。

## 测试过程（手动）

### A. 等比预览是否正常
步骤：
1. 打开 index.html → 进入编辑器
2. 打开任意场景编辑 → 切到 Step-4「编辑对话」
3. 观察右侧预览画面

预期：
- 预览画面应居中显示，不会被“横向拉满后变成一条细长条”。
- 预览区域会出现合理黑边（letterbox），保证 16:9 画面完整。

实际：
- 静态检查：editor.html 已将 iframe 包裹在 .editor-preview-viewport，并由 __fitPreviewViewport() 按 16:9 动态计算 viewport 宽高；不会再强行把 iframe 拉到容器的全部宽高。

验收点：
- 你看画面时，能看到完整的背景/人物/对白框，而不是只露出上/下的一截。

### B. 分隔条拖拽调整比例
步骤（宽屏左右布局）：
1. 保持窗口宽度 > 1020px
2. 在 Step-4 拖动中间的分隔条（鼠标指针应为左右调整）

预期：
- 右侧预览区域宽度实时变化。
- 预览画面始终等比缩放，内容不被裁切。

实际：
- 静态检查：已实现 pointer 事件拖拽逻辑（__initDialogSplitter），拖动时会写入 ww_editor_previewPaneW。

验收点：
- 你能把预览拖宽/拖窄，并且右侧不会“突然撑满又看不见”。

步骤（窄屏上下布局）：
1. 缩小窗口宽度 ≤ 1020px（触发上下布局）
2. 拖动分隔条（鼠标指针应为上下调整）

预期：
- 右侧预览区域高度实时变化。
- 预览画面仍保持 16:9，且内容完整。

实际：
- 静态检查：已实现 ww_editor_previewPaneH 保存与高度限制（最大 0.85vh）。

验收点：
- 你能把预览拖高一点，不再被固定 520px 卡死。

### C. 尺寸持久化
步骤：
1. 在 Step-4 拖拽分隔条调整到你喜欢的比例
2. 关闭场景编辑器 → 再打开同一个/任意场景 → 切到 Step-4

预期：
- 预览区域的宽/高保持你上次设置。

实际：
- 静态检查：localStorage 读写键已存在（ww_editor_previewPaneW / ww_editor_previewPaneH），并在 openSceneEditModal / updateStepUI / window.resize 时应用。

验收点：
- 你不用每次都重新调比例。

## 备注
- 这里的“实际”我只能做到代码级静态验收（环境里没法替你打开浏览器点来点去）。如果你跑起来发现某个浏览器对 pointer capture 特别矫情，直接告诉我具体浏览器版本和现象，我再补兼容。
