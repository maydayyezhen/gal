# D-UI-01 场景编辑器全屏布局（解决拥挤/看不见）

## 改动内容
- 场景编辑弹窗改为占满视口的全屏工作区（不再是居中小盒子）。
- 顶部新增工具条：显示当前场景ID +「浏览器全屏」按钮 + 关闭按钮。
- 步骤条改为横向可滚动的紧凑样式，减少挤压。
- 内容区改为按步骤独立滚动（避免出现双滚动条）。
- Step-4（对话编辑）
  - 左侧对话列表可伸缩占满高度，内部滚动。
  - 「+ 添加新对话」按钮底部吸附。
  - 右侧预览框高度随视口自适应，不再写死 540px。
  - 窄屏（<=1020px）自动切换为上下布局。
- Step-5 场景预览高度上限提高到 55vh。

## 测试过程
1. 用浏览器打开 `game_project/editor.html`。
2. 选择任意章节，点击某个场景的“编辑”。
3. 检查场景编辑器是否占满当前窗口，并且顶部工具条与底部按钮栏始终可见。
4. 切换到 Step-4：
   - 左侧对话列表可滚动，右侧预览框保持可见且高度适配窗口。
   - 「+ 添加新对话」按钮在滚动到底部时仍然容易找到。
5. 将浏览器窗口宽度缩小到约 1000px：Step-4 应变为上下布局。
6. 点击「浏览器全屏」按钮进入全屏，再按 ESC 退出，按钮文案应跟随切换。
7. 打开“查看改动”（diff）等其它弹窗，确认其仍然是原来的居中样式（不被强行全屏）。

## 预期 / 实际结果（验收点）
- 验收点1：场景编辑器在 1920x1080 / 1366x768 下没有内容被遮挡。实际：通过（布局改为全屏+滚动）。
- 验收点2：Step-4 左列表与预览框不互相挤压，且可各自滚动/适配高度。实际：通过（列表 flex+overflow，预览框高度自适应）。
- 验收点3：窄屏自动适配，不需要水平滚动才能看见控件。实际：通过（<=1020px 切上下）。
- 验收点4：全屏按钮可进可退（ESC 也能退）。实际：通过（监听 fullscreenchange 同步文案）。

备注：这里的“实际”是基于代码结构与样式约束的验证 + JS 语法检查（node --check），不包含每种浏览器的逐像素视觉回归测试。
